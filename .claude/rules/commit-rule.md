# Commit Message Rules

Use conventional commits with Korean descriptions.
Prefix with scope when clear: `feat(backend):`, `fix(ui):`, `chore(tauri):`.

| Type       | Description   |
| :--------- | :------------ |
| `feat`     | New feature   |
| `fix`      | Bug fix       |
| `docs`     | Documentation |
| `refactor` | Refactoring   |
| `test`     | Tests         |
| `chore`    | Build/config  |

Keep first line under 72 characters.
