The secrets manager that understands your dependency graph.

Stores keys. But also tells you who uses them, where, and what breaks if you revoke. Auto-matches NVD/GHSA incidents to your vault. Scans your npm/Cargo deps for known secret-leak history. Free. Open source. Runs fully offline.

What you get on day one — free, no account

Dependency graph

Visual map of Issuer → Credential → Usage → Project → Deployment → URL. Filter, search, simulate.

Blast radius

Preview which services break before you revoke a key. No production accidents.

Incident feed

NVD / GHSA / issuer RSS polled locally and matched to your vault. Server never sees what you store.

Supply chain scan

OSV.dev queried for every npm / Cargo dep with secret-leak history. Lockfile-aware.

RAILGUARD

Auto-generates .cursorrules / CLAUDE.md / Copilot instructions so AI editors can’t exfiltrate keys.

CLI + MCP + IDE

apivault in your terminal. MCP server for Claude / Cursor. VS Code & JetBrains plugins. M18 · M21 · M22

Vault Charter recovery

Diceware 6 words + 4-digit verifier. Optional Shamir 2-of-3 split for inheritance. Forget your passphrase, not your vault. M23

Why this exists

Existing vaults answer "where is the key?" — none answer "what happens when this key leaks?" or "is the npm package I'm about to install going to phone home with my .env?" That gap is where breaches start. We close it on your laptop, before the breach mail arrives.
