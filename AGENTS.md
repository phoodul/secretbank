# Project Rules (Cross-Tool)

## Language
- Respond in Korean. Code and commit messages in English.

## Code Style
- Simplicity first. No single-use abstractions.
- Do not add features beyond what was asked.
- Match existing patterns in the codebase.

## Verification
- Always verify changes with tests.
- Address root causes, not symptoms.

## Git
- Use conventional commits with scope: feat(backend):, fix(ui):.
- Never commit secrets or credentials.
